<RichText id="overview" title="The Mental Model">

Agent-Native Plans has two important shapes of the same plan:

- **Runtime plan**: normalized JSON stored in SQL. This is what the visual editor reads and writes while you are reviewing.
- **Source plan**: repo-friendly MDX files such as `plan.mdx`, `canvas.mdx`, `prototype.mdx`, and `.plan-state.json`.

The hosted Plan app keeps the runtime plan in hosted SQL. The local Plan app can also mirror every saved runtime edit back to MDX files in your repo. That is the useful autosync path today: visual/editor edits -&gt; action save -&gt; SQL -&gt; local MDX mirror.

</RichText>

<Columns id="two-directions" title="The Two Directions Are Not Equal Yet">
<Column id="visual-to-file" label="Works Now">

<Callout id="visual-to-file-copy" title={"Visual -> file"} tone="success">

When the local Plan app is running, browser/agent saves go through Plan actions. Those actions persist SQL and then call the local file mirror. This is the autosync path that already exists.

</Callout>

<Code
  id="visual-to-file-code"
  title="Local run shape"
  language="bash"
  caption="The exact port may differ if the dev server chooses another free port."
  code={
    "PLAN_LOCAL_ODE=1 PLAN_LOCAL_DIR=/path/to/repo/plans pnpm dev\nagent-native skills add visual-plan --mcp-url http://localhost:8105"
  }
/>

</Column>

<Column id="file-to-visual" label="Manual Today">

<Callout id="file-to-visual-copy" title={"File -> visual"} tone="warning">

Editing `plan.mdx` directly does not automatically update the browser/database. The supported path is `import-visual-plan-source` for a folder, or `patch-visual-plan-source` for surgical MDX patches.

</Callout>

<Code
  id="file-to-visual-code"
  title="Source-action shape"

  caption="This keeps runtime JSON normalized after source edits."
  code={

  }
/>

</Column>

<Callout id="decision" title="Recommendation" tone="decision">

Treat local Plan SQL/JSON as canonical while the editor is running, and MDX as a mirrored source artifact. Add a small CLI wrapper for local plan sessions before adding a file watcher. A watcher can come later, but it needs conflict detection so browser autosave and editor saves do not overwrite each other.

</Callout>
