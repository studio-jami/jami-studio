<RichText id="overview">

## Recommendation

Make **hosted Plans** the default experience. A normal user should install the skill, ask for `/visual-plan`, review in the hosted Plan app, and never clone or run the Plan app locally.

Separate the product decision into two independent axes:

- **Where the Plan app runs:** hosted, local packaged runtime, or custom/self-hosted MCP URL.
- **How files sync:** none, manual export/import, hosted repo writeback, or local MDX mirror.

That keeps the docs honest: localhost is not the mainstream product path; it is an advanced mode for offline work, framework development, and teams that explicitly want local files as the primary artifact.

## Decisions from review

- Build toward **hosted + repo sync** first, so people get files without running localhost.
- Make mode choice **flags only** for now; skip a guided picker in the first pass.
- Start with **one-way visual -&gt; files** writeback plus explicit import/patch for file edits.

<RichText id="heading-two-axes">

## The model to document

Explain runtime location and file sync separately so hosted usage stays simple.

</RichText>

<Columns id="two-axes" title="Two Separate Choices">
<Column id="runtime-location" label="Plan Runtime">

This chooses where the editor, comments, review events, and normalized JSON live.

- **Hosted:** simplest, always current, shareable.
- **Local packaged:** no account/network required, writes local files.
- **Custom URL:** organization-controlled deployment.

<Callout id="runtime-callout" tone="decision">

Docs should say hosted is the default and local is advanced. Do not present cloning the Plan app as the ordinary install path.

</Callout>

</Column>

<Column id="file-sync" label="File Sync">

This chooses whether MDX artifacts are produced.

- **None:** runtime only; export when needed.
- **GitHub writeback:** hosted plan saves produce repo files through a trusted integration.
- **Local mirror:** local saves write to `PLAN_LOCAL_DIR`.
- **Import/patch:** explicit file-to-runtime updates with conflict checks.

<Callout id="sync-callout" tone="warning">

Avoid invisible two-way sync at first. Browser autosave and editor saves need revision ids and conflict handling before they can safely run both directions live.

</Callout>

</Column>
</Columns>

<Callout id="localhost-reality" title="How localhost works today" tone="info">

Today, --mcp-url can point a skill at a local Plan app, but that local app must already be running. In this repo it runs from the working tree. In a productized version, the CLI should run a packaged @agent-native/core@latest local Plan server so people do not clone the app. That still is not the default hosted path; it is an explicit local/offline session.

</Callout>
