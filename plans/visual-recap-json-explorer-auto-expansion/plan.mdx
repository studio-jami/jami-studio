<RichText id="recap-compatibility" title="Behavior compatibility">

- **JsonExplorerRead default collapsedDepth**: changed (risky). Before:
  `data.collapsedDepth ?? 1`. After:
  `data.collapsedDepth ?? JSON_EXPLORER_DEFAULT_COLLAPSED_DEPTH (2)`. Existing
  blocks with an explicit collapsedDepth are unchanged; blocks omitting it now
  open one additional container level.
- **JsonExplorerEdit auto-expand presets**: added (non-breaking). After: `Off, 2
levels, 3 levels, All`. The existing numeric input remains for custom depth.
- **JsonNode disclosure click**: added (non-breaking). After: `Alt/Option-click
expands or collapses all rendered descendants for that subtree`. Regular click
  still toggles only the clicked node.
- **jsonExplorerSchema.data.collapsedDepth.max**: changed (neutral). Before:
  `max(20)`. After: `max(JSON_EXPLORER_MAX_COLLAPSED_DEPTH)`.

</RichText>

<Columns id="recap-before-after">
<Column id="before" label="Before" contentId="before-text">

JSON explorer blocks opened only the root container by default. The edit popover exposed a raw `Collapsed depth` number input. Disclosure rows only toggled their own node.

</Column>

<Column id="after" label="After" contentId="after-text">

JSON explorer blocks open two container levels by default. The edit popover has Off / 2 levels / 3 levels / All presets plus a custom number input. Alt/Option-clicking a disclosure row deeply expands or collapses that subtree.

</Column>

<RichText id="recap-risk" title="Review notes">

- Main compatibility note: the only behavior change for existing data is when `collapsedDepth` is omitted; that now defaults to 2 instead of 1.
- The deep toggle is intentionally local to the clicked subtree. Global **Expand all** / **Collapse all** still work through the root pulse.
- The recap is grounded in the current git diff, but it does not certify correctness. Review the changed lines, especially the recursive pulse/remount behavior in `JsonNode`.
