# Jami Studio Landing Page Canon

## Positioning

Jami Studio builds adaptable agent frameworks for real workspaces: software where humans, agents, UIs, apps, and external tools share the same actions, SQL state, and context.

Jami Studio is for builders who want agent-powered products without surrendering their stack. Models, databases, storage, search, analytics, deployment, and observability remain replaceable. Apps can run locally, deploy anywhere the framework supports, and evolve from a single harness into a full workspace suite.

## Product Thesis

The next generation of agent software is not chat alone. It is agent and interface parity.

Every meaningful operation is defined once as an action. A person can trigger that action through a button, form, voice command, automation, or keyboard shortcut. An agent can call the same action as a tool. External agents can reach it through MCP or A2A. The result lands in durable application state, where every surface can see and continue the work.

Jami Studio turns agents into collaborators inside real software, not transcripts beside it.

## Core Principles

**Provider agnostic by default**
Jami Studio is built around replaceable provider lanes. Teams can choose their own model providers, SQL databases, search systems, storage/CDN providers, analytics tools, observability stack, and deployment targets.

**One action surface**
Actions are the shared contract across UI, agent, HTTP, MCP, A2A, CLI, automation, and voice. The same operation powers every surface.

**SQL-backed shared state**
Application state lives in SQL so agents and interfaces see the same world. Agent work updates the UI. User actions update agent context.

**Installable registry**
Jami Studio distributes harnesses and apps through an installable registry. Builders can start with one harness, one app, a curated suite, or a complete workspace.

**Intentional workspace suites**
Apps are grouped around real modes of work: business, design, research, and coding. Each suite can be installed whole or assembled a la carte.

**Central orchestration**
The orchestration app connects the workspace. It owns shared resources, vaults, inboxes, approvals, routing, and cross-app delegation so specialist apps can stay focused.

**Voice-first real-time interaction**
Jami is a natural-language machine interface for directing work as agents operate in the background. Voice is the primary real-time interaction layer, with optional video/avatar presence for continuity, collaboration, and richer feedback. Voice, chat, app UI, and automation all route into the same action and orchestration system.

## Product Surfaces

**Harnesses**
Harnesses are agent-control surfaces and runtime integrations: coding agents, browser sessions, visual planning, PR recaps, MCP/A2A bridges, voice interaction, and other ways to give agents a safe operating environment.

**Apps**
Apps are functional, installable workspaces with real UI and durable data. Examples include planning, mail, calendar, forms, analytics, content, design, assets, slides, clips, and orchestration.

**Suites**
Suites bundle apps and harnesses for a coherent workflow.

- Business: mail, calendar, forms, analytics, content, orchestration.
- Design: design, assets, slides, clips, visual planning.
- Research: knowledge capture, source review, reports, content databases, external data connectors.
- Coding: visual plans, PR recaps, code agents, browser debugging, harness integrations.

**Jami**
Jami is the voice-first real-time interface to the workspace. It lets a person talk, interrupt, approve, redirect, inspect, and continue work while agents run in the background.

**Intercal**
Intercal is a supporting knowledge-delta project for model training-cutoff supplementation and discovery. It is related to the Jami Studio ecosystem, but it is not a technical dependency of the Jami Studio agent framework.

## Audience

Jami Studio is for solo developers, technical founders, and small teams building agent-enabled internal tools, workflow systems, and product surfaces.

It also serves teams already using coding agents, MCP tools, workflow automation, design systems, research workflows, or analytics tools who want those workflows to become durable, inspectable software.

   Build apps where agents and humans share the same actions, state, and context. Install one harness, one app, or a full workspace suite.

2. **Agent and UI parity**
   Every button can be a tool. Every tool can have a UI. Voice, chat, app screens, automation, MCP, and A2A all meet at one action surface.

   Talk to Jami while agents work in the background. Use voice for flow, full UI for precision, and optional avatar/video presence for richer real-time collaboration.

4. **Registry and suites**
   Install harnesses, individual apps, or curated workspace suites for business, design, research, and coding.

5. **Central orchestration**
   One inbox, one vault, one resource layer, one router for specialist apps.

6. **Adapt your stack**
   Bring your providers. Swap models, databases, search, storage, analytics, observability, and deployment targets as your product evolves.

7. **Supporting ecosystem**
   Intercal extends the broader ecosystem with knowledge-delta workflows for current information and discovery.
