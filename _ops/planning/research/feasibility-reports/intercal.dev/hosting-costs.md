# Hosting And Service Cost Matrix

Date: 2026-05-21
Status: Planning reference

Prices are planning estimates and should be verified before account setup. Provider pricing changes frequently, especially compute, model APIs, and managed database plans.

## Hosting Paths

| Service Area             |                 Local Development Path |                                           Low-Cost Hosted Path |                                                                         Production Managed Path |
| ------------------------ | -------------------------------------: | -------------------------------------------------------------: | ----------------------------------------------------------------------------------------------: |
| App/API/MCP compute      |            Local machine, Docker, `$0` |             Single VPS or small app host, typically `$4-10/mo` |                                    Managed app host, typically `$10-50+/mo` before usage growth |
| Python workers/scheduler |                    Local machine, `$0` |           Same VPS, `$0 extra`; or small scheduled worker host |                    Serverless/scheduled workers or separate worker host, typically `$10-50+/mo` |
| Postgres + pgvector      |                  Docker Postgres, `$0` |             Same VPS Postgres, `$0 extra`; or Neon Free/Launch |                                     Managed Postgres, often `$15-70+/mo` depending load/storage |
| Redis/queue/cache        |              Docker Redis/Valkey, `$0` |        Same VPS Redis/Valkey, `$0 extra`; or Upstash Free/PAYG |                             Managed Redis/queue, typically `$0-25+/mo`, higher with SLA/add-ons |
| Object storage           |        Local filesystem or MinIO, `$0` |                    Cloudflare R2 free tier likely enough early | Cloudflare R2 or S3-compatible store, usually low single digits until storage/read volume grows |
| Embeddings               | Local model, `$0`; or hosted API usage | Local model if VPS has enough RAM; otherwise hosted embeddings |      Hosted embeddings or dedicated worker; variable, expected low early but workload-dependent |
| LLM extraction/synthesis |                         API usage only |                                                 API usage only |                                                    API usage only; likely largest variable cost |
| Domain/DNS               |                               Optional |                  Domain about `$10-20/yr`; Cloudflare DNS free |                                                                                            Same |
| Email/notifications      |                        Usually skipped |                   Free/low-volume transactional tier if needed |                                           Paid transactional email if notification volume grows |
| Monitoring/logs          |                       Local logs, `$0` |                  VPS logs plus optional free hosted monitoring |                           Hosted monitoring/log retention, `$0-80+/mo` depending retention/team |

## Recommended Posture

Use all three paths as first-class deployment targets:

- Local development path for reproducible work and tests.
- Low-cost VPS path for a cheap real public instance and dogfooding.
- Managed production path when uptime, backups, scale, and operational clarity justify the spend.

The architecture should keep these paths substitutable. Intercal should depend on service contracts, not a specific hosting provider.
