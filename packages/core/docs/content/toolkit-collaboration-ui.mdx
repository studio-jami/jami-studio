pieces that do not own transport or storage:

```tsx
import {
  AgentPresenceChip,
  LiveCursorOverlay,
  PresenceBar,
  RecentEditHighlights,
  RemoteSelectionRings,
} from "@agent-native/toolkit/collab-ui";
```

Core still owns the live collaboration runtime: Yjs documents, awareness sync,
presence hooks, recent edit publication, undo grouping, and server plugins. Use
core for data and Toolkit for rendering.

```tsx
import { useCollaborativeDoc, usePresence } from "@agent-native/core/client";
import { PresenceBar } from "@agent-native/toolkit/collab-ui";
```

Existing imports from `@agent-native/core/client` still work because core
re-exports these Toolkit components as compatibility shims.

See the [Collaboration Kit](/docs/toolkit-collaboration) for the full runtime,
presence, and undo contract.

## Responsibilities {#responsibilities}

Toolkit owns:

- Presence avatars and agent editing chips.
- Live cursor overlays.
- Remote selection rings.
- Recent edit highlight overlays.
- Display helpers such as collaborator color and display-name derivation.

Core owns:

- Yjs document creation and sync.
- Awareness polling and SSE fast-path transport.
- Collaboration server plugins and actions.
- Recent edit publication and collection from awareness.
