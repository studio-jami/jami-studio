# Toolkit

Toolkit es la estantería de bloques reutilizables y probados para aplicaciones
agénticas de alta calidad. Convierte el trabajo de producto que se repite entre
apps en superficies estándar, sin poner un patrón de aplicación en el centro del
catálogo.

Esta sección es el único lugar para descubrir UI reutilizable al crear apps. Una
superficie pertenece a estos documentos porque es reutilizable, incluso si hoy
la implementa Core o una plantilla. Las etiquetas de propiedad y las rutas de
importación hacen explícito ese límite sin dispersar la guía por el sitio.

Each kit packages the UI patterns, action contracts, app-state expectations,
access boundaries, and agent-visible behavior that users expect to work
consistently. Apps still own their domain model and visual identity; Toolkit
make the common product systems feel complete from the first version instead of
being rebuilt in slightly different ways every time.

## Madurez y propiedad de paquetes {#package-maturity-and-ownership}

- El editor enriquecido es propiedad de **Toolkit**; los imports existentes de Core siguen siendo reexportaciones de compatibilidad ligeras durante la migración.
- Design es un **patrón de plantilla** con `design-tweaks` reutilizables, no un editor ni lienzo completo listo para usar.
- Blocks y el motor Context X-Ray siguen siendo propiedad de **Core** en esta fase; la presentación reutilizable de Context X-Ray es de Toolkit.
- Creative Context, Scheduling y Pinpoint son **módulos de capacidad de Toolkit** instalados bajo demanda; conservan paquetes npm y manifiestos de ciclo de vida separados. Dispatch sigue siendo un producto separado.

## Límite de implementación {#implementation-boundary}

Los kits son primero superficies de producto. Algunas piezas usan el runtime
(`@agent-native/core`), some are presentation primitives
(`@agent-native/toolkit`), and apps usually wrap both through local adapters.
Usa la ruta del paquete que posee el comportamiento necesario, pero diseña las
experiencias de ajustes, colaboración, compartir, equipo, revisión y
configuración como un kit coherente, sin exponer límites de paquetes en la app.

## Piezas fundamentales {#foundation-pieces}

- [Primitivas de UI](/docs/toolkit-ui) cubre las primitivas compartidas estilo
  shadcn bajo `@agent-native/toolkit/ui/*`, el patrón de adaptador local de cada
  plantilla y los hooks de shell compartidos.
- [API de componentes](/docs/components) sigue siendo la referencia de nivel
  inferior para los exports públicos de React.

## Regla de diseño {#design-rule}

Usa un kit cuando el comportamiento se repite entre apps y los usuarios esperan
que sea estándar: compartir, ajustes, equipos, comentarios, historial y
progreso del agente no deberían variar entre plantillas. Conserva en la app los
datos, flujos, etiquetas y estilo visual específicos del dominio.
