# Paketlebenszyklus

Wiederverwendbare Pakete können ein statisches, versioniertes
`agent-native.package.json` veröffentlichen. Die CLI liest nur diese JSON-Datei
und führt beim Prüfen keinen Paketcode aus.

```bash
agent-native package inspect @agent-native/scheduling --json
agent-native package add @agent-native/scheduling --apply
agent-native package eject @agent-native/scheduling --apply
```

Hinzufügen und Auswerfen sind standardmäßig Vorschauen. Erst `--apply` schreibt
Dateien und startet den erkannten Paketmanager. Kollisionen und unbekannte
Layouts werden abgelehnt; bei einem Installationsfehler werden Dateien und
Lockdateien zurückgesetzt. Zunächst unterstützt nur Scheduling das Auswerfen;
Pinpoint bleibt bis nach dem Pilotprojekt zurückgestellt.

## Migrationen

Veröffentlichte Pakete enthalten `migration-manifest.json` für verschobene Pfade und Symbole. `npx @agent-native/core@latest upgrade --codemods` zeigt den Diff; `--yes` wendet ihn nach Prüfung an. In CI: `agent-native doctor --only migration-manifest`, ohne Opt-out.

Einträge mit `planned` können vor ihrem Ziel veröffentlicht werden; sie erscheinen als nicht blockierende Warnungen und werden nicht umgeschrieben. Ein aktiver Move wird nur umgeschrieben, wenn der Ziel-Export installiert ist.
