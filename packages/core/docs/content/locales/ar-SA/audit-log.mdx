# سجل التدقيق

يحصل كل تطبيق وكيل أصلي على سجل تدقيق خارج الصندوق: سجل متين وكامل ومخصص للوصول وملحق فقط **من قام بتغيير بيانات التطبيق ومتى وأين و- متى كان الوكيل - وفي أي تشغيل.** يتم الالتقاط تلقائيًا في خط الإجراء؛ لم تكتب أي رمز لذلك.

نظرًا لأن الوكيل يمكنه تغيير البيانات نيابةً عنك، فإن السؤال الرئيسي الذي يجيب عليه سجل التدقيق هنا ليس فقط "من قام بتحرير هذا السجل" - بل **"هل كان هذا أنا أم الوكيل، وما هو السبب وراء ذلك؟"** لا يمكن لأي نظام آخر في إطار العمل الإجابة على ذلك.

## التدقيق مقابل إمكانية الملاحظة مقابل التتبع {#which}

تسجل ثلاثة أنظمة "ما حدث" لثلاثة أسباب مختلفة. اختر السؤال الذي تطرحه:

| النظام                                   | السؤال الذي يجيب عليه                               | الإخلاص                            | الجمهور                       |
| ---------------------------------------- | --------------------------------------------------- | ---------------------------------- | ----------------------------- |
| **سجل التدقيق** (هذه الصفحة)             | "من قام بتغيير هذا السجل، ومتى، وهل كان هو الوكيل؟" | **كامل ومتين وذو نطاق محدد**       | المستخدم، المشرف، الوكيل نفسه |
| **[Observability](/docs/observability)** | "لماذا فعل الوكيل ذلك، وما هي التكلفة؟"             | قياس المدى عن بعد                  | المطور                        |
| **[Tracking](/docs/tracking)**           | "كيف يستخدم الأشخاص المنتج؟"                        | أطلق النار وانسى إلى SaaS الخارجية | PM / النمو                    |

يعد سجل التدقيق الذي يتم أخذ عينات منه أو شحنه إلى موفر التحليلات عديم الفائدة - بيت القصيد هو أنه كامل ومحلي وقابل للاستعلام عنه. لذا فهو نظام فرعي خاص به، وليس نمطًا من النظامين الآخرين.

## ما يتم التقاطه تلقائيًا {#captured}

عند تشغيل أي إجراء **متغير** (أي شيء ليس `GET` للقراءة فقط)، يُلحق إطار العمل صفًا واحدًا بـ `agent_audit_log` مع:

- **الإجراء** — اسم الإجراء (على سبيل المثال، `delete-recording`).
- **الممثل** — `agent`، أو `human`، أو `system`، بالإضافة إلى البريد الإلكتروني للممثل — تتم ملؤه **حتى بالنسبة لمكالمات الوكيل**، بحيث تحصل على "الوكيل، الذي يعمل لصالح alice@، ...".
- **تشغيل الرابط** — العامل `threadId` / `turnId` الذي قام بتشغيل الاستدعاء (استدعاء أداة)، لذا فإن الطفرة تتتبع مرة أخرى إلى المنعطف الدقيق للوكيل.
- **السطح** — `tool` (الوكيل)، أو `frontend`، أو `http`، أو `cli`، أو `mcp`، أو `a2a`.
- **النتيجة** — `success`، أو `error` (مع رمز خطأ)، أو `denied` (تم حظره بواسطة بوابة موافقة بشرية).
- **المدخلات** — وسيطات الاستدعاء، بقيم على شكل بيانات الاعتماد [redacted](#privacy).
- **الهدف والمالك** — المورد الذي تم تغييره بواسطة الإجراء، والمستخدم في [scope reads](#reading).

لا حاجة إلى أسلاك — يتم ربط خطافات الالتقاط بـ `defineAction` بشفافية. يتم تخطي actions للقراءة فقط، ويتم تخطي بعض إطارات العمل عالية التردد actions (مزامنة حالة التطبيق، وأشعة السياق السينية، والتنقل) افتراضيًا لتجنب إغراق السجل.

## أعلن عن الإجراء الذي تغير {#target}

افتراضيًا، يتم تحديد نطاق الحدث إلى **الممثل** — حيث ترى تغييراتك وتغييرات الوكيل نيابةً عنك. لإجراء تغيير على مورد _shared_ يظهر أيضًا في مسار **المالك**، ولتسمية الأحداث حسب المورد، قم بإعلان `target`:

```ts
export default defineAction({
  description: "Delete a recording.",
  schema: z.object({ id: z.string() }),
  audit: {
    target: (args, result) => ({
      type: "recording",
      id: args.id,
      // Optional — defaults to the actor. Set when editing someone else's resource.
      ownerEmail: result?.ownerEmail,
      visibility: "org",
    }),
    summary: (args) => `Deleted recording ${args.id}`,
  },
  run: async (args, ctx) => {
    /* ...delete... */
  },
});
```

كل شيء في `audit` اختياري. الحد الأدنى للإضافة المفيدة هو `target: () => ({ type, id })`.

### ضبط الالتقاط {#tuning}

| الخيار               | التأثير                                                 |
| -------------------- | ------------------------------------------------------- |
| `audit.target`       | قم بتسمية الحدث بالمورد والنطاق الذي تمت قراءته لمالكه. |
| `audit.summary`      | سطر قصير يمكن قراءته للحدث.                             |
| `audit.onRead`       | تدقيق **قراءة** حساسة (الوصول السري، التصدير المجمع).   |
| `audit.enabled`      | القبض على قوات `true`؛ يختار `false` طفرة صاخبة.        |
| `audit.recordInputs` | يتخطى `false` التقاط الوسائط (المنقحة بالفعل).          |

## قراءة المسار {#reading}

تتوفر ثلاث قراءات actions للوكيل **و** الواجهة الأمامية في كل تطبيق، ويتم تحديد نطاقها في SQL للمتصل - ولا تُرجع أبدًا صفوف مستأجر آخر:

- **`list-audit-events`** — التصفية حسب `targetType` / `targetId`، `actorKind` (`agent` | `human` | `system`)، `status`، `threadId` / `turnId`، `action`، `sinceMs`، و`limit`.
- **`get-audit-event`** — حدث واحد حسب المعرف، بما في ذلك حمولة الإدخال المنقحة الخاصة به.
- **`export-audit-events`** — تصدير السجل بالجملة بصيغة CSV أو NDJSON بدلاً من ترقيم `list-audit-events` يدويًا، لعمليات السحب دون اتصال لأغراض الامتثال. تأخذ نفس عوامل تصفية `list-audit-events` (بدون `limit`) بالإضافة إلى `format` (`csv` | `ndjson`، الافتراضي `csv`) و`maxRows` (الافتراضي 5000، الحد الأقصى 10000)، وتُرجع `{ content, rowCount, truncated, format }` — تعني `truncated: true` أن عدد الصفوف المطابقة تجاوز `maxRows`. وهي نفسها قراءة حساسة، يتم تدقيقها عبر `audit: { onRead: true }`.

أنشئ خلاصة نشاط أو سطر "من قام بتغيير هذا" عن طريق استدعاء `list-audit-events` من UI باستخدام `useActionQuery` — لا تكتب أبدًا يدويًا أي جلب إلى جدول التدقيق:

```tsx
import { useActionQuery } from "@agent-native/core/client";

const { data } = useActionQuery("list-audit-events", {
  targetType: "recording",
  targetId: recordingId,
});
// data.events → [{ action, actorKind, actorEmail, turnId, status, summary, createdAt }, …]
```

يمكن للوكيل استدعاء نفس الإجراء - اسأله "ما الذي قمت بتغييره في هذا التسجيل؟" ويجيب من الدرب.

## الخصوصية والاحتفاظ {#privacy}

- **التنقيح** — قبل تخزين أي مدخلات، تتم إزالة المفاتيح والقيم على شكل بيانات الاعتماد (الرموز المميزة، والأسرار، وكلمات المرور، والسلاسل الحاملة) واقتطاع الحمولات كبيرة الحجم. لا يصبح سجل التدقيق أبدًا مخزنًا ثانويًا للأسرار. احتفظ بنص `summary` خاليًا من البيانات الحساسة أيضًا.
- **إلحاق فقط** — لا يوجد إجراء تحديث أو حذف لصفوف التدقيق. الحذف الوحيد هو عملية تطهير الاحتفاظ، مما يجعل السجل جديرًا بالثقة باعتباره مسارًا للتدقيق.
- **عزل المستأجر** — يتم تحديد نطاق القراءات وفقًا لهوية المتصل ومؤسسته؛ بدون هوية، لا شيء يطابق.

التكوين عبر البيئة:

- `AGENT_NATIVE_AUDIT_RETENTION_DAYS` — مدة الاحتفاظ بالصفوف (افتراضي `365`؛ `0` = الاحتفاظ للأبد).
- `AGENT_NATIVE_AUDIT_ENABLED=false` — مفتاح الإيقاف العام.
